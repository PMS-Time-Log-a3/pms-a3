Mobile Inventory Control

Inventory records in one place

This Ionic application helps users manage inventory records through a mobile-friendly interface. Each record stores the required item name, category, quantity, price, supplier name, stock status, featured item value, and optional special note.

Connected to the assessment API

The app communicates with the remote REST endpoint to list all records, search for a single item by name, create new records, update existing records, and delete records when allowed by the server.

Clear stock information

Items are organised with stock status values such as In Stock, Low Stock, and Out of Stock, making it easier to understand availability and identify records that may need attention.

Validation before submission

The form checks required fields before sending data. Quantity and price must be whole numbers, while category and stock status must be selected from the allowed values.

Separate pages for key tasks

The List page shows all records and supports editing or deleting. The Add page creates new records. The Search page finds one item by name. The Featured page displays records marked as featured.

Using Inventory Data Responsibly

Shared database notice

The app works with a remote assessment database. Records created in the app may be visible through the same endpoint, so item details should be appropriate and related to inventory work.

Accurate item names

Item names are unique and are used for searching, updating, and deleting. Users should check spelling before saving a new item or changing an existing record.

Limit sensitive information

Inventory fields should describe products only. Supplier names and special notes should not contain passwords, payment details, private contact information, or unrelated personal information.

Careful update and delete actions

Changes are sent directly to the server. Users should review the selected item carefully before confirming edits or deletion because removed records may not be recoverable from the app.

Device safety

When using a shared computer or mobile device, users should close the browser after finishing so another person does not accidentally continue editing inventory records.

Possible future improvements

The app could be improved with user sign-in, role-based permissions, and an audit history for important changes.
